Domain Models, Events, Command Flow

Die stabile Mitte der Spine: was gespeichert wird, was als Event fließt und wie Aktionen das System durchlaufen

Primary Domain Models

Workspace

id, name, root_path, connection_target, default_layout_id, tags, metadata

Lokaler Arbeitskontext. In V1 immer local, aber mit remote-fähigem Zielmodell.

Session

id, workspace_id, name, status, active_tab_id, focused_panel_id, snapshot_ref, timestamps

Beschreibt eine fortsetzbare UI-/Arbeits-Sitzung, nicht einen magisch serialisierten Prozessbaum.

Layout

id, name, tabs[], split_tree, panel_bindings, focus_path

Rekonstruiert Tabs, Splits, Größen und Fokus ohne UI-spezifische Widget-Interna zu speichern.

PanelState

panel_id, panel_type, config, snapshot, persist_policy

Jedes Panel besitzt sein eigenes Snapshot-Schema und seine eigene Persistenzgrenze.

Command

id, source, name, args, context, timestamp

Einheitliche Repräsentation für Slash, Palette, Keybinding und Panel-Aktion.

Event Types

Domain Events

  • WorkspaceOpened
  • SessionCreated
  • SessionRestored
  • LayoutApplied
  • CommandExecuted
  • SnapshotSaved

Runtime / UI Events

  • PanelMounted
  • PanelFocused
  • PTYStarted
  • ProcessOutputReceived
  • TerminalExited
  • StatusMessagePublished
Persist vs Recreate
  • Persist: active workspace, tabs, split ratios, focused panel, cwd, explorer selection, command history
  • Persist: WorkPanel config and terminal launch settings
  • Recreate: PTY handle, process id, open stream buffers, interactive subprocess state
