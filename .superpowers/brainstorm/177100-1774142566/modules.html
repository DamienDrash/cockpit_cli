Core Modules

Welche Teile der Spine im ersten Spec real existieren und wie sie zusammenspielen

Module Map

AppShell

Root Textual app. Hält Header, Tabbar, Statusbar, Command Palette, Slash Input und aktive Navigation.

WorkspaceService

Lädt Workspace-Kontext, Root Path, Default Layout und Startparameter für den ersten Screen.

SessionService

Erstellt, lädt, snapshotet und restauriert Sessions. Kennt die Resume-Grenze.

LayoutService

Baut Tabs, Split-Struktur, Panel-Platzierung und Fokuspfad aus gespeicherten Layoutdaten.

CommandDispatcher

Zentrale Aktionsschicht für Slash, Palette, Keybindings und UI-Aktionen.

EventBus

Lose Kopplung zwischen Shell, Services, Panels und Status-UI über typisierte Events.

PanelHost + WorkPanel

Panel-Lifecycle, Snapshot/Restore und die Referenzimplementierung für Dateikontext plus eingebettetes Terminal.

LocalShellAdapter + PTYManager

Lokale Shell starten, cwd setzen, Output streamen, Terminal-Lebenszyklus überwachen.

SQLiteStore + ConfigLoader

Sessions, Workspaces, Layouts und History in SQLite; deklarative Layouts/Bindings/Themes aus YAML/TCSS.

Must Exist in First Spec
  • Typed domain models for Workspace, Session, Layout, PanelState, Command
  • One dispatcher for all action sources
  • Panel lifecycle contract
  • SQLite-backed snapshot persistence
  • One real local runtime path via PTY
Explicitly Deferred
  • SSH execution and remote rehydration
  • Git/Docker/DB/Curl/Cron feature panels
  • Live process resume across app restarts
  • Plugin API exposure to third parties
