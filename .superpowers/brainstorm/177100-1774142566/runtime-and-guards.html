Runtime Boundaries, Error Handling, Guardrails

Was der lokale Runtime-Pfad garantieren muss und wie die Spine bei Fehlern sichtbar und kontrolliert bleibt

Local Runtime Contract

PTYManager

Startet lokalen PTY, setzt cwd/env, liefert resize/stdout/exit Events, beendet Prozesse kontrolliert.

StreamRouter

Leitet Runtime-Output an den eingebetteten Terminal-Bereich und an Status-/Audit-Kanäle weiter.

TaskSupervisor

Überwacht Hintergrundjobs wie Snapshot-Writes, config reloads oder health checks.

Filesystem Adapter

Validiert Workspace- und cwd-Pfade, listet Verzeichnisse, liefert Metadaten für das WorkPanel.

Failure Modes
  • Workspace path missing
  • Stored cwd invalid
  • PTY creation fails
  • Snapshot write fails
  • Layout snapshot incompatible with current schema
Expected Behavior
  • No crash loops on startup
  • Visible recovery state inside AppShell
  • Actionable status message with retry / reset path
  • Session remains inspectable even if terminal runtime failed
  • Corrupt snapshots can be ignored or reset safely
Guardrails in the First Slice

Included

  • Safe path validation before workspace open
  • Explicit recovery UI instead of silent fallback
  • Audit metadata for session open, restore, snapshot save, terminal start/exit
  • Schema/version field on snapshots

Deferred

  • Prod/stage risk coloring for remote targets
  • Destructive-action confirmations for Docker/DB/Cron panels
  • Secret store integration beyond local-ready interfaces
  • Remote transport health/reconnect policy
