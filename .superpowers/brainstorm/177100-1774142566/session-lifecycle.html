Session, Snapshot, Resume, Panel Lifecycle

V1-Resume stellt Arbeitskontext zuverlässig wieder her, ohne falsche Versprechen über laufende Prozesse zu machen

Session Snapshot Layers

Light Snapshot

Häufig gespeichert: aktiver Tab, Fokus, cwd, Explorer-Auswahl, Panel-Flags, letzte Commands.

Structural Snapshot

Bei Layoutänderungen: Tabs, Split-Ratios, Panel-Bindings, Sichtbarkeit, Fokuspfad.

Resume Snapshot

Beim Beenden/Suspend: Workspace-Ref, Layout-Ref, Panel-Snapshots, Terminal-Launch-Konfiguration, keine Live-PTY-Objekte.

Resume Flow
Load Session Row
Resolve Workspace
Apply Layout
Instantiate Panels
Restore PanelState
Recreate Terminal in stored cwd
Publish SessionRestored

If workspace path is missing or invalid, resume degrades to a clear recovery state instead of crashing.

WorkPanel Lifecycle
  • mount() panel host created and wired
  • initialize(context) workspace path, selection, launch config injected
  • restore_state(snapshot) explorer/work-context restored
  • attach_terminal() fresh PTY started in stored cwd
  • snapshot_state() returns persistable panel data only
  • suspend() terminal stopped or detached cleanly
  • dispose() event subscriptions and runtime handles released
Explicit Non-Goals
  • No resurrection of arbitrary interactive subprocesses
  • No replay of transient terminal output buffers across restarts
  • No hidden best-effort reattachment that can silently lie to the user
  • Failure must surface as visible session recovery state
